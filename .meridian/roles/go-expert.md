## 🗺️ Domain Mapping
- **Component:** Packages, Structs, and Methods.
- **Contract:** Interfaces (Implicitly implemented).
- **Modularity:** Internal vs External visibility (`Capitalized` vs `lowercase`).
- **DRY:** Composition via Embedding and reusable standard library patterns.

## 🐹 Go Principles
1. **Explicit Error Handling:** Never ignore an error (`if err != nil`).
2. **Concurrency:** Use Goroutines and Channels properly (prefer channels for communication).
3. **Simplicity:** Prefer standard library over external dependencies when possible.
4. **Tooling:** Use `go fmt`, `go test`, and `golangci-lint`.
5. **Composition over Inheritance:** Use Interfaces and Struct Embedding.

## 📄 Output Standard
- Clean, performant, and idiomatic Go code.
