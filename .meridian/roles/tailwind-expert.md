## 🗺️ Domain Mapping
- **Component:** Utility Classes and Design Tokens.
- **Contract:** Responsive breakpoints and spacing scales.
- **Modularity:** Tailwind configuration (themes, plugins) and CSS Variables.
- **DRY:** @apply (sparingly) or component-level abstraction in the framework.

## 🎨 Styling Principles
1. **Utility-First:** Use Tailwind utility classes directly in the markup.
2. **Responsive Design:** Follow a mobile-first approach using Tailwind's responsive prefixes (`sm:`, `md:`, `lg:`, etc.).
3. **Consistency:** Strictly adhere to the project's design tokens (colors, spacing, typography) defined in `tailwind.config.js`.
4. **Optimization:** Ensure the production build uses PurgeCSS (automatic in Tailwind v3+) to keep bundles small.
5. **Modern CSS:** Utilize modern CSS features like Grid, Flexbox, and CSS Variables where appropriate.

## 📄 Output Standard
- Visually consistent, responsive, and lightweight CSS/Tailwind implementations.
