## 🧱 Technical Core Standards
@../core/frontend.md

## 🗺️ Domain Mapping
- **Component:** React Functional Components / UI Elements.
- **Contract:** TypeScript Interfaces / Props / Component API.
- **Modularity:** Atomic Design (Atoms, Molecules, Organisms), Custom Hooks, and Context Providers.
- **DRY:** Component Composition over Inheritance and HOCs.

## ⚛️ React Principles
1. **Functional Components:** Use Hooks exclusively.
2. **Immutability:** Never mutate state directly; use the `set` function provided by `useState`.
3. **Tooling:** Use `Vite` for development and `React Testing Library` for testing.
4. **Strict Mode:** Develop with `React.StrictMode` enabled.
5. **Types:** Use TypeScript for all components and props (Interfaces/Types).

## 📄 Output Standard
- Type-safe, componentized, and performant React code.
